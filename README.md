# jira-reports

Набор скриптов на Python для выгрузки данных из **Jira Server / Data Center** (REST API v2, Basic Auth) по проекту **ERPWE** и последующей обработки CSV/Excel для отчётов и биллинга.

---

## Пошаговая инструкция: с нуля до первого запуска

Ниже — что сделать человеку, который **не обязан знать Git и Python**. Команды ниже вводятся в **терминале** (окно с текстом и приглашением вроде `PS C:\...>`). На Windows удобно использовать **PowerShell**.

### Шаг 1. Получить папку с файлами проекта

Нужна одна папка, внутри которой лежат скрипты (`export_erpwe_comments.py`, `requirements.txt` и т.д.).

**Вариант А — вам прислали архив (ZIP и т.п.)**

1. Сохраните архив на диск (например в «Загрузки»).
2. Распакуйте архив: правый щелчок по файлу → **Извлечь все…** (или через 7-Zip / встроенный проводник).
3. Запомните путь к **внутренней** папке, где лежат `.py` файлы — это **корень проекта**. Пример: `C:\Users\Имя\Documents\jira-reports`.

**Вариант Б — есть Git и доступ к репозиторию**

1. Установите [Git for Windows](https://git-scm.com/download/win), если его ещё нет.
2. Откройте PowerShell в пустой папке, куда хотите положить проект.
3. Выполните команду клонирования (подставьте свой URL репозитория):

   ```text
   git clone <URL-репозитория>
   cd jira-reports
   ```

   Дальше «корень проекта» — это текущая папка после `cd`.

### Шаг 2. Установить Python

1. Зайдите на [python.org/downloads](https://www.python.org/downloads/) и скачайте установщик для Windows (**Python 3.10 или новее**).
2. Запустите установщик.
3. На первом экране **включите галочку** «Add python.exe to PATH» / «Добавить Python в PATH» и нажмите **Install Now**.
4. Дождитесь окончания установки и при необходимости перезагрузите компьютер (если команда `python` всё равно не находится — закройте и снова откройте терминал).

**Проверка:** откройте PowerShell и выполните:

```powershell
python --version
```

Должна появиться строка с версией, например `Python 3.12.x`. Если пишет, что команда не найдена, переустановите Python с галочкой PATH или используйте ярлык **«Python 3.x»** из меню Пуск и запускайте команды через **«Python Command Prompt»**, открыв его в папке проекта (см. шаг 3).

### Шаг 3. Открыть терминал в папке проекта

Терминал должен быть **в корне проекта** (там же, где `requirements.txt`).

**Через проводник Windows**

1. Откройте папку проекта в проводнике.
2. Щёлкните по адресной строке (где путь к папке), введите `powershell` и нажмите Enter — откроется PowerShell уже в этой папке.

**Или вручную**

1. Откройте PowerShell.
2. Перейдите в папку командой (подставьте свой путь):

   ```powershell
   cd C:\путь\к\jira-reports
   ```

### Шаг 4. (Рекомендуется) Виртуальное окружение

Так зависимости не смешиваются с другими программами на компьютере.

В PowerShell **в корне проекта** выполните по очереди:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

После активации в начале строки часто появляется префикс `(venv)`.

Если PowerShell пишет про **политику выполнения** (execution policy), один раз выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

и снова `.\venv\Scripts\Activate.ps1`.

**Если виртуальное окружение не используете** — просто пропустите шаг 4; команды `python` и `pip` ниже всё равно выполняйте из папки проекта.

### Шаг 5. Установить зависимости проекта

В том же терминале (с активированным `venv` или без него):

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Дождитесь сообщений без ошибок в конце (успешная установка `requests`, `python-dotenv`, `openpyxl`).

### Шаг 6. Файл с паролем к Jira (`.env`)

1. В папке проекта найдите файл `.env.example`.
2. Сделайте **копию** и переименуйте копию в **`.env`** (именно такое имя, без `.txt` на конце). В проводнике Windows при переименовании иногда получается `.env.txt` — тогда в Проводнике включите показ расширений файлов и уберите лишнее `.txt`.
3. Откройте `.env` в **Блокноте** и подставьте свои значения:
   - `JIRA_BASE_URL` — адрес вашей Jira;
   - `JIRA_USER` — логин;
   - `JIRA_PASSWORD` — пароль или токен.

Сохраните файл. **Не отправляйте `.env` в чаты и не вкладывайте в письма** — в нём секреты.

### Шаг 7. Запустить первый скрипт

Терминал по-прежнему должен быть открыт **в корне проекта** (и с `(venv)`, если вы его активировали).

Пример — выгрузка полного отчёта в файл `erpwe_report.csv`:

```powershell
python export_erpwe_comments.py -o erpwe_report.csv
```

Если всё настроено верно, скрипт поработает и создаст CSV в этой же папке. Если появится текст об ошибке — прочитайте сообщение (часто это неверный URL, логин/пароль или сеть).

Дальнейшие команды — в разделе **«Типовой порядок работ»** ниже на этой странице.

## Кратко для опытных

- Python **3.10+**, `pip install -r requirements.txt`.
- Переменные окружения: см. `.env.example`; при желании задайте `JIRA_JQL` для дефолтного JQL в `export_erpwe_comments.py`.
- `.env` в корне проекта; в Git не коммитится.

## Скрипты

| Скрипт | Назначение |
|--------|------------|
| `export_erpwe_comments.py` | Выгрузка задач по JQL: комментарии с домена `@beringpro.ru` в заданном окне дат, ворклоги по задаче, эвристика «минут на комментарий». Пишет CSV (по умолчанию `erpwe_beringpro_comments.csv`). |
| `export_erpwe_worklogs.py` | Выгрузка ворклогов в CSV с фильтром по времени начала ворклога (`--worklog-from` / `--worklog-to`, обязательны). |
| `build_short_report.py` | Сжатый отчёт из полного CSV: одна строка на комментарий BeringPro; разделитель `;`, UTF-8 с BOM. Вход по умолчанию: `erpwe_report.csv`. |
| `issue_authors_from_report.py` | Список авторов (имя + email) по каждому `issue_key` из полного отчёта. |
| `analyze_billing_support.py` | Сопоставление списка на выставление (`erpwe_short_report.xlsx` или `.csv`) с полным текстом из `erpwe_report.csv`; эвристики «не поддержка» (ЗНИ, оценки, гарантия и т.д.). Результат: Excel с листами `не_поддержка`, `все_строки_с_меткой`, `методология`. |

Клиент Jira и загрузка `.env` реализованы в `export_erpwe_comments.py`; `export_erpwe_worklogs.py` переиспользует тот же клиент.

## Типовой порядок работ

1. **Полный отчёт из Jira** (задайте JQL и окно комментариев при необходимости):

   ```powershell
   python export_erpwe_comments.py -o erpwe_report.csv
   ```

   По умолчанию выход — `erpwe_beringpro_comments.csv`; для шагов ниже удобно сразу указать `-o erpwe_report.csv`, как в `build_short_report.py`.

2. **Короткий отчёт для биллинга**:

   ```powershell
   python build_short_report.py -i erpwe_report.csv -o erpwe_short_report.csv
   ```

3. **Проверка строк на «не поддержка»** (нужен `openpyxl`; для `.xlsx` биллинга — без конвертации):

   ```powershell
   python analyze_billing_support.py --billing erpwe_short_report.xlsx --jira erpwe_report.csv
   ```

4. **Ворклоги за период** (отдельная выгрузка):

   ```powershell
   python export_erpwe_worklogs.py --worklog-from 2026-03-01 --worklog-to 2026-04-30 -o erpwe_worklogs.csv
   ```

5. **Авторы по задачам** из полного CSV:

   ```powershell
   python issue_authors_from_report.py -i erpwe_report.csv -o erpwe_issue_authors.csv
   ```

## Примечания

- Для HTTPS с самоподписанным сертификатом можно передать `--no-verify-ssl` (осторожно: снижает безопасность).
- Сгенерированные `.csv` и `.xlsx` в `.gitignore` — артефакты хранятся локально.
